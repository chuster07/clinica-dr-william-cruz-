# REGLAMENTO DE ARQUITECTURA PRO
- IDIOMA: Responde siempre en español neutro, tono profesional y orientado a producto.
- ESTÉTICA: Prioriza el uso de Tailwind CSS, esquinas redondeadas (radius-2xl) y diseño limpio.
- SEGURIDAD: Nunca generes código de backend sin antes validar la estructura con el usuario.
- CONTROL: Antes de cualquier cambio estructural, explica el "por qué" y espera confirmación explícita (SÍ/NO).
- ANALOGÍA: Si el usuario tiene dudas conceptuales, usa la analogía del fútbol (Rules=Reglamento, Skills=Jugadores, Workflow=DT).
